#Git spurningar og svör
1. Hvağ gera eftirfarandi Linux skipanir?
	cd >> færir şig í directoryiğ sem şú velur(şarft ağ segja hvert şú vilt fara t.d cd git_verkefni sendir şig í şann folder
	cd ~ >> færir şig upp um eitt directory
	ls >> sınir şér allt sem er í şví directory sem şú ert í 
	pwd >> sınir şér hvar şú ert staddur
	mkdir >> Bır til folder 

2. Hvağ gera eftirfarandi git skipanir og hvers vegna eru şær gagnlegar?
	git clone >> copyar eğa clonar innhald stağsettningar og setur şağ í nıtt directory, gagnlegt til şess ağ hafa aauğveldan ağgang ağ şví sem şú varst  ağ sækjast eftir
	git log >> sınir şér allt şağ sem şú ert búinn ağ vera ağ commita, mjög gott ağ geta fariğ yfir vinnuna sem şú ert búinn ağ gera ef şú hefur ekki t.d skrifağ skrefinn niğur
	git status >> sınir şér leiğir sem eru öğruvísi eins og t.d ef şú myndir commita meğ şví ağ setja git commit en hitt myndi sına şér hvağ ef şú hefğir commitağ meğ şví ağ runna git add fyrst. mjög gott ağ geta séğ hvağ gæti veriğ öğruvísi.
	git diff >> leiğ til şess ağ bera saman tvær mismunandi leiğir. sem er mjög gott til şess ağ sjá muninn greinilega.
	git checkout >> uppfærir  filana í şeirri leiğ sem şú ert ağ vinna í şanning ağ şær séu sama útgáfa eins og şær í indexinu eğa í şeirri leiğ sem şú velur, ef şú velur enga leiğ şá mun şağ uppfæra allt, gott til şess ağ fljótlega hafa allt í sömu útgáfu.

3. Hver er munurinn á eftirfarandi git skipunum:
	a) git diff şetta er til şess ağ bera saman mismunandi foldera, greinar eğa leiğir.
	b) git diff --staged >> şetta er til şess ağ sjá allt sem şú ert búinn ağ gera tilbúiğ fyrir commit
	c) git diff commit1 commit2 >> şett er til şess ağ bera saman tvö commit
